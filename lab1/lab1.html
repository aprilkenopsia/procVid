1. Introdução

Este relatório descreve o primeiro laboratório de Processamento de Vídeo, sendo uma introdução à biblioteca OpenCV. Os objetivos incluíram a configuração do ambiente, o carregamento de imagens de arquivos, a captura de vídeo via webcam e a gravação dos resultados em disco. Este documento detalha os procedimentos adotados e os resultados obtidos.

2. Fundamentos Básicos

O modelo de processamento de imagem e vídeo se baseia em um fluxo de Entrada/Saída (E/S). Este fluxo é composto por três etapas: entradas, processamento e saídas.

  • Entradas (Inputs): Corresponde à forma como o programa obtém os dados visuais, que podem vir de duas fontes principais: fontes ao vivo (câmeras) ou arquivos (imagens e vídeos).
  • Processamento (Processing): Etapa em que o programa, utilizando a biblioteca OpenCV, atua sobre os dados de entrada, podendo aplicar e filtros e realizar transformações.
  • Saídas (Outputs): Destino dos dados após o processamento, que pode ser a exibição em tela (monitor) ou a gravação em um novo arquivo de imagem ou vídeo.

As atividades do laboratório foram executadas para implementar diferentes combinações deste fluxo.

3. Métodos

  • Parte 1, Preparação do Ambiente: A primeira etapa consistiu na configuração do ambiente de desenvolvimento. O procedimento seguiu o guia L1_parte1_instalar_opencv_v25.txt, que orientou a instalação de dependências e a compilação da biblioteca OpenCV a partir do código-fonte.
  • Parte 2, Estudo do Processamento Básico: Nesta etapa, foram estudados os programas de exemplo para as operações de E/S.
    • (A) Leitura de Imagem: Utilizando o tutorial Using OpenCV with gcc and CMake o programa de exibição de imagem foi compilado e executado para carregar e exibir o arquivo messi5.jpg.
    • (B) Leitura e Gravação de Vídeo: Os cinco programas em C++ do tutorial Reading and Writing Videos using OpenCV foram compilados e executados para análise de suas funcionalidades.
  • Parte 3, Obtenção de Fotos e Vídeos: A terceira parte focou na aplicação dos conceitos estudados para a geração de arquivos de mídia.
    • Captura de Foto: O programa video_write_from_webcam.cpp foi modificado para que a captura fosse de uma única imagem, e não de uma sequencia de frames, gerando o programa img_write_from_webcam.cpp. Ao pressionar “q”, o frame atual do vídeo era salvo como um arquivo de imagem.
    • Gravação de Vídeos: Foi utilizado o programa video_write_from_webcam.cpp, estudado na Parte 2, sem modificações, para gravar os quatro vídeos solicitados.

4. Resultados e Análises

Esta seção apresenta os resultados obtidos com os procedimentos descritos.

4.1. Resultados do Estudo Básico (Parte 2)

  • Análise dos Programas de Vídeo: A análise dos cinco programas resultou na seguinte descrição de suas funções e aplicações:
    • video_read_from_file.cpp: Lê e exibe um vídeo a partir de um arquivo. Aplicação: base para análise de vídeos pré-gravados.
    • video_read_from_image_sequence.cpp: Cria um vídeo a partir de uma sequência de imagens. Aplicação: criação de timelapses e animações.
    • video_read_from_webcam.cpp: Captura e exibe o vídeo de uma webcam em tempo real. Aplicação: base para sistemas interativos.
    • video_write_from_webcam.cpp: Captura, exibe e salva o vídeo de uma webcam. Aplicação: software de gravação.
    • video_write_to_file.cpp: Lê um vídeo de um arquivo e salva em outro. Aplicação: conversão de formato ou salvamento de resultados após processamento.
